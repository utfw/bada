# Project BADA (바다)

모바일 웹 기반 3D 해양 체험 프로젝트

## 1. 목표

- 정면, 후면, 위, 아래면에서 볼 수 있는 360도 가상 해양 공간 구현
- 휴대폰의 자이로 센서와 연동하여 시점이 자연스럽게 변경
- 현재 날씨에 따라 해양 환경이 변화 (외부 API 사용)
- 고래가 헤엄쳐서 지나가는 애니메이션 연출

## 2. 기술 스택

| 분류 | 기술 |
|------|------|
| 언어 | TypeScript |
| 3D 렌더링 | Three.js |
| 빌드 도구 | Vite |
| 날씨 API | OpenWeatherMap API |
| 자이로 센서 | DeviceOrientation Web API |
| 3D 모델 | glTF / GLB 포맷 |
| 배포 | Vercel 또는 Netlify |

## 3. 핵심 기능

### 3.1 3D 해양 공간

- 사용자가 수중에 있는 듯한 360도 환경
- 바닥: 해저 지형 (모래, 바위, 해초)
- 위: 수면 + 하늘 (빛이 수면을 통해 내려오는 효과)
- 주변: 파티클로 표현되는 부유물, 기포

### 3.2 자이로 센서 시점 제어

- `DeviceOrientationEvent`를 활용한 카메라 회전
- alpha(좌우), beta(상하), gamma(기울기) 값 매핑
- 자이로 센서 미지원 기기: 터치 드래그로 시점 변경 (fallback)
- PC 환경: 마우스 드래그로 시점 변경 (fallback)

### 3.3 날씨 연동 환경 변화

| 날씨 상태 | 해양 환경 변화 |
|-----------|---------------|
| 맑음 | 밝고 맑은 수중, 햇빛 광선(god rays)이 수면을 통해 내려옴 |
| 흐림 | 어둡고 탁한 수중, 시야 거리 감소 |
| 비 | 수면에 빗방울 파동 효과, 약간 어두운 조명 |
| 눈 | 차가운 푸른 톤, 수중에 미세한 흰 파티클 |
| 안개 | 안개 효과(fog)로 시야 크게 감소 |

- 사용자 위치 기반 현재 날씨 자동 반영 (Geolocation API)
- 위치 권한 거부 시 서울 기본값 사용

### 3.4 고래상어 애니메이션

- 프로시저럴 생성(LatheGeometry + 커스텀 지오메트리) — GLB 외부 에셋 의존 없음
- **종**: 고래상어(Whale Shark) — 세계에서 가장 큰 어류
- **크기**: SHARK_LENGTH 14 유닛, 사용자가 압도감을 느낄 수 있는 수준
- **형태**: 편평한 몸체(Y축 0.75 스케일), 수직 heterocercal 꼬리지느러미(상엽 > 하엽), 프로미넌트 등지느러미, 긴 paddle 형 가슴지느러미, 5쌍 아가미 슬릿
- **색상**: 회청색 바탕(#3a4e63) + 흰 반점 격자 패턴(whale shark signature)
- **꼬리 움직임**: 좌우 스윕(상어 특징) — 고래의 상하 움직임과 반대
- **몸체 웨이브**: 꼬리 쪽으로 진폭이 증가하는 S자 sin 곡선 좌우 변위
- **경로**: CatmullRomCurve3 3D 곡선, 진행 방향으로 자연스럽게 회전 (lookAt)
- 멀리서 다가와 사용자 근처를 지나가는 연출

### 3.5 물고기 군집 (Fish Schooling)

- **Boids 알고리즘**: 분리(Separation) + 정렬(Alignment) + 응집(Cohesion) 3가지 규칙으로 군집 유영 동작 구현
- **개별 물고기**: 저폴리 메시, 꼬리 지느러미 sin 진동 애니메이션 포함
- **군집 경로**: 완만한 타원 경로를 따라 씬 내부 순환
- 고래와 물고기가 함께 등장해 씬의 생동감을 높임

## 4. UI / UX

### 4.1 로딩 화면

- 3D 에셋 로딩 진행률 표시
- "화면을 터치하여 시작" 안내 (자이로 권한 요청 트리거)

### 4.2 HUD (Heads-Up Display)

- 현재 날씨 상태 아이콘 (좌측 상단, 최소한의 UI)
- 위치 정보 텍스트 (도시명)

### 4.3 인터랙션

- 화면 탭: 고래 등장 트리거
- 자이로/드래그: 시점 변경

## 5. 프로젝트 구조

```

├── index.html
├── src/
│   ├── main.ts              # 진입점

│   │   ├── Lighting.ts       # 조명 및 날씨별 효과

│   ├── controls/
│   │   └── DeviceControls.ts # 자이로 / 터치 / 마우스 입력 처리
│   ├── weather/
│   │   └── WeatherService.ts # 날씨 API 호출 및 데이터 가공
│   ├── ui/
│   │   ├── LoadingScreen.ts  # 로딩 화면
│   │   └── HUD.ts            # 날씨 아이콘, 위치 표시
│   └── utils/
│       └── constants.ts      # 상수 정의
├── public/
│   └── models/               # 고래 등 3D 모델 파일
├── design.md
├── tsconfig.json
├── vite.config.ts
└── package.json
```

## 6. 성능 고려사항

- 모바일 GPU 최적화: 저폴리곤 모델, 텍스처 압축
- `requestAnimationFrame` 기반 렌더 루프
- 화면 밖 오브젝트 렌더링 제외 (frustum culling)
- 에셋 lazy loading으로 초기 로딩 최소화

## 7. 포트폴리오 어필 포인트

- **기술적 도전**: Three.js를 활용한 실시간 3D 렌더링
- **센서 활용**: 디바이스 하드웨어(자이로) 연동 경험
- **외부 API 연동**: 날씨 API와 Geolocation 활용
- **반응형 대응**: 자이로 → 터치 → 마우스 단계적 fallback
- **몰입감 있는 UX**: 날씨에 따라 달라지는 동적 환경
